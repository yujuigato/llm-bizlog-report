Background

這個系統在解決什麼問題？

業務人員每天寫日報、做紀錄，但這些寶貴的客戶洞察都散落在非結構化文字中，無法系統性地彙整為可行動的業務情報。

😣 過去的痛點

  • 17 萬筆業務日誌散落在 CRM，無法快速分析
  • 企業痛點、需求、洞察靠人工整理，極度耗時
  • 無法識別哪些企業有高價值需求但尚未跟進
  • 業務無法知道同產業客戶的共同痛點在哪

✅ 系統如何解決

  • LLM 自動萃取每家企業的 8 大屬性與 7 層銷售情報
  • ML 聚類為 7 種法人類型，找出共同痛點模式
  • 四路整合生成痛需熱圖與業務推薦卡片
  • 訂單時間序列分析，追蹤成交路徑與再購行為
Pipeline

四階段分析管道

從原始日誌到可行動情報，每個 Phase 各自解決不同層次的問題。

Phase 1

基礎屬性萃取

Gemini 2.5 Flash 從業務日誌萃取 8 大法人基礎屬性，覆蓋 178,790 家企業

Phase M

法人類型聚類

Scikit-learn 依屬性分群為 7 種法人類型（C0–C6），識別共同痛點模式

Phase 2

七層情報萃取

LLM 萃取 L1–L7 銷售情報層（痛點 → 需求 → 洞察 → 解決方案），覆蓋 173,376 家

Phase 3 / 4

整合與視覺化

四路資料整合生成痛需熱圖、業務推薦卡片；訂單時間序列 Sankey / Sunburst 圖

Intelligence Layers

L1–L7 七層銷售情報

每家企業從業務日誌中萃取七個層次的情報，從表面痛點到深層洞察。

層級內容意義
L1企業現象描述業務觀察到的表面問題
L2業務猜測原因業務對問題根因的推斷
L3客戶表達痛點客戶直接陳述的困擾
L4客戶未說出的痛點行間讀出的隱性需求
L5客戶需求/目標客戶明確期望達到的狀態
L6業務洞察業務從互動中獲得的深層理解
L7解決方案線索可對應產品/服務的具體切入點
